Report

Template
Instead of using a template, we used semantic-ui which is a css framework. The fremework allowed us to build well-designed and simple interface.

Models
We used sqllite and web2py DAL(database abstraction layer) to define our tables.

Dynamic Content
We tried to make the pages dynamic as much as possible. The only static pages are Homepage and About page.

In our site, films and directors are the main items that have dynamic structure. Comments and genres are the other items that have also dynamic structure.


User Management
We basically created a user login, logout and register system by using web2py's own user tables that allowed more advanced usage. For example; by using the decorator @auth.requires_login() we restricted access to the functions to only logged in users.

Tables
We used semantic-ui tables to get well-formed views in the dynamic contents such as films.

Forms
In our site, we used forms to create an object or edit an existing object. Actually, we did all the CRUD(create, read, update and delete) operations somewhere in the project.

Additional Data
In the project, we have films and directors as main items. Additionally, we have genres, comments and watchlist that have relations to the main items.

Data Insertion
First, we populated the data by hand in the lists(in default.py file) then we inserted all the data programatically using the lists. In this way, we avoided the data loss.

Additional Functionality
In the project, we used jquery and javascript to get more functionality such as search.
